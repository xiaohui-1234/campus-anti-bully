# 开发者接口字段说明书

本文按当前代码实现整理，覆盖后端、小程序、配置后台、MQTT、WebSocket、数据库、缓存和对象存储字段。所有对外 JSON 字段均以 `snake_case` 为准。

## 1. 系统边界

### 后端

目录：`campus-server`

职责：

- 微信小程序登录与 JWT 鉴权。
- 用户资料和头像上传。
- 设备绑定、设备资料维护和在线状态查询。
- 报警事件查询、标记已读、删除、音频访问 URL 刷新。
- MQTT 消息接入、去重、异步分发和下发上传指令。
- MinIO 预签名上传/访问 URL 生成。
- WebSocket 事件和设备状态推送。
- ADMIN 后台运行配置和系统信息接口。

### 微信小程序

目录：`campus-miniprogram`

页面：

- 登录：`pages/login`
- 首页：`pages/home`
- 事件：`pages/events`
- 设备：`pages/devices`
- 状态：`pages/status`
- 个人：`pages/profile`

小程序统一通过 `services/request.js` 访问 HTTP，通过 `services/websocket.js` 建立 `/ws/v1/client` 单连接。

### 配置后台

目录：`campus-admin-web`

页面：

- `/login`
- `/config`
- `/cache`
- `/storage`
- `/mqtt`
- `/mqtt-test`
- `/system`

后台普通登录接口走 `/api/v1`，配置接口走 `/backend/v1`，后端要求 `ADMIN` 角色。

### STM32 主控

目录：`stm32主控源码/联网测试win`

当前实现：

- ESP8266 WiFi/MQTT/HTTP 上传。
- TF 卡 FatFs 文件系统与 SD SPI 驱动。
- WAV 录音文件上报。
- MQTT 在线心跳、报警上报、上传确认。
- 报警类型包括按钮报警和多种语音关键词报警。
- `TF_Card_Reset()` 为软复位：卸载文件系统、重新初始化 SD SPI、重新挂载。

## 2. 全局字段规则

### 命名

- HTTP JSON 字段：`snake_case`。
- WebSocket 消息字段：`snake_case`。
- MQTT payload 字段：`snake_case`。
- Java 内部 DTO/VO 使用 camelCase，由 Jackson 统一转为 `snake_case`。

### 时间

- HTTP 响应时间格式：`yyyy-MM-dd HH:mm:ss`。
- 时区：`Asia/Shanghai`。
- MQTT `timestamp` 使用毫秒时间戳。

### 敏感字段

不得明文保存或返回：

- `openid`
- `device_secret`
- `wifi_password`
- JWT access token / refresh token 日志明文
- MinIO `secret_key`
- 微信 `app_secret`

允许保存：

- `openid_hash`
- `device_secret` 哈希
- `file_key`
- `wifi_name`

## 3. HTTP 通用返回字段

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | number | `0` 表示成功，非 0 表示失败 |
| `message` | string | 提示信息 |
| `data` | object/array/string/null | 业务数据 |

## 4. Auth 字段

### `POST /api/v1/auth/wx/login`

请求：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `code` | string | 是 | 微信 `wx.login` 返回的临时 code |

响应 `data`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `access_token` | string | JWT access token |
| `refresh_token` | string | JWT refresh token |
| `token_type` | string | 固定 `Bearer` |
| `expires_in` | number | access token 剩余秒数 |
| `user_info` | object | 用户信息 |

### `POST /api/v1/auth/openid/admin-login`

请求：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `openid` | string | 是 | 开发管理员 openid |

该接口用于开发后台登录。生产环境应替换为更安全的管理员授权流程。

### `POST /api/v1/auth/refresh`

请求：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `refresh_token` | string | 是 | refresh token |

响应：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `access_token` | string | 新 access token |
| `token_type` | string | `Bearer` |
| `expires_in` | number | 剩余秒数 |

### `POST /api/v1/auth/wx/logout`

请求 Header 必须带 access token。后端将 token ID 写入 Redis 黑名单。

| 分组 | 字段 |
| --- | --- |
| `cache` | `mqtt_dedup_ttl_seconds`、`device_online_ttl_seconds`、`bind_code_ttl_seconds`、`access_url_ttl_seconds` |

### ConfigUpdateRequest

当前支持更新：

- `cache`
- `storage`

MQTT broker、账号、MinIO endpoint 等基础设施连接参数当前只读展示，变更后通常需要重启服务。

## 13. 验收要点

- 所有接口返回字段为 `snake_case`。
- 小程序能登录、绑定设备、查看事件、播放音频、标记已读、删除事件。
- 首页在线设备数量按 `online_status=ONLINE` 统计。
- 事件卡片和详情中 `VOICE/fight` 显示为 `声音/打架`，未知值原样显示。
- 设备报警后后端创建事件，并下发 `alarm/upload`。
- 设备上传成功后发布 `alarm/confirm`，事件 `file_status` 更新为 `SUCCESS`。
- WebSocket 只使用 `/ws/v1/client`，订阅多个设备时校验绑定关系。
- Redis 去重和数据库唯一索引均生效。
- 文档、日志、响应均不暴露敏感明文。
