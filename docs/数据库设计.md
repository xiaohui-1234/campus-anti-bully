# 数据库设计

数据库名：`campus`。

核心表：

- `device`：设备身份信息，`device_secret` 只保存哈希。
- `user`：微信用户信息，只保存 `openid_hash`。
- `user_device_bind`：用户与设备绑定关系，保存 `device.id` 与 `user.id`。
- `event`：报警事件，保存 `device_table_id`、`event_id`、文件状态、推送状态、阅读状态。
- `device_config_wifi`：设备 WiFi 配置列表，不保存 WiFi 密码。

完整建表脚本见 `sql/campus.sql`。
