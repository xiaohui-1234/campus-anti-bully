# 部署说明

## 环境依赖

- JDK 17+
- Maven 3.9+
- MySQL 8
- Redis
- MinIO
- EMQX 或兼容 MQTT Broker
- Node.js 20+
- 微信开发者工具
- STM32/Keil 环境用于主控固件编译烧录

## 数据库初始化

```bash
mysql -uroot -p < sql/campus.sql
```

脚本会创建 `campus` 数据库和核心表。根目录脚本包含开发种子设备和管理员，生产环境请替换或删除。

## 后端部署

```bash
cd campus-server
mvn clean package
java -jar target/campus-server-1.0.0.jar
```

生产环境必须外部覆盖以下敏感配置：

- `spring.datasource.url`
- `spring.datasource.username`
- `spring.datasource.password`
- `spring.data.redis.host`
- `spring.data.redis.password`
- `campus.security.jwt.secret`
- `campus.wx.miniapp.app-id`
- `campus.wx.miniapp.app-secret`
- `campus.minio.endpoint`
- `campus.minio.access-key`
- `campus.minio.secret-key`
- `campus.mqtt.broker-url`
- `campus.mqtt.username`
- `campus.mqtt.password`

不要把真实密钥提交到仓库。

## 后台部署

```bash
cd campus-admin-web
npm install
npm run build
```

构建产物在 `dist`。部署时需要让 `/api/v1`、`/backend/v1`、`/ws/v1/client` 反向代理到后端服务。

## 小程序部署

1. 使用微信开发者工具打开 `campus-miniprogram`。
2. 修改 `config/env.js`：

```js
baseUrl: 'https://your-domain.example/api/v1'
wsUrl: 'wss://your-domain.example/ws/v1/client'
```

3. 配置微信小程序合法域名：
   - request 合法域名：后端 HTTPS 域名。
   - socket 合法域名：WebSocket WSS 域名。
   - upload/download 合法域名：后端和对象存储公网域名。

## EMQX/MQTT

后端通过 `campus.mqtt.*` 连接 Broker。设备侧使用 ESP8266 AT 指令连接同一 Broker。topic 必须保持：

```text
device/{product_type}/{device_id}/{action}
```

硬件联调前确认：

- Broker TCP 端口对设备可达。
- 后端能连接 Broker。
- 设备 MQTT 用户名/密码与 Broker 认证配置一致。
- 后端 `campus.mqtt.enabled=true`。
- EMQX 客户端列表中出现 `campus-server` 表示后端 MQTT 客户端已连接，这是正常现象。
- 同一 `campus.mqtt.client-id` 同一时间只能被一个后端实例使用。多个实例都使用 `campus-server` 时会互相顶掉连接，后端日志会反复出现 `MQTT connection lost`、`MqttException: 已断开连接`、`EOFException`。

当前后端 MQTT 消费使用普通通配订阅，没有使用 shared subscription。生产或测试环境建议只启动一个 `campus.mqtt.enabled=true` 的后端实例；如需多开后端实例做 HTTP/WebSocket 联调，其他实例应关闭 MQTT，或临时使用不会与主实例冲突的 `client-id` 并避免同时消费设备上报 topic。

## MinIO

后端需要能访问 MinIO `endpoint`，设备需要能访问预签名上传 URL 对应的公网地址。若设备只能访问 HTTP，应确保 `upload_url` 的协议、host、端口对 ESP8266 可达。

## STM32 主控

1. 修改 `stm32主控源码/联网测试win/Hardware/campus_config.h` 中 WiFi、EMQX、设备 ID、产品类型、设备 MQTT 认证等配置。
2. 使用 Keil 打开 `Project.uvprojx` 编译烧录。
3. 确认 TF 卡、麦克风/录音、ESP8266、按键和供电稳定。
4. 设备启动后应连接 WiFi、连接 MQTT、订阅 `alarm/upload`、周期上报 `status/online`。
