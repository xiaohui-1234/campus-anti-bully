
统一接口形式
{
  "code": 0,
  "message": "success",
  "data": {}
}

code = 0     成功
code = 1     业务失败
code = -1    参数/系统异常
code = 401   未登录或 token 失效
code = 403   无权限

/---------------------------------------------------------------/

后台路径前缀 /backend/v1/
api路径前缀  /api/v1/



用户页面：
POST    /api/v1/auth/refresh                       用刷新Token自动刷新JWTToken
POST    /api/v1/auth/wx/login                      小程序登录接口
POST    /api/v1/auth/wx/logout                     小程序登出接口

GET     /api/v1/users/me                           获取个人信息，头像url
PUT     /api/v1/users/me                           修改个人信息
POST    /api/v1/users/me/avatar                    上传个人头像



消息页面：
GET     /api/v1/events/unpulled                    补拉所有设备未送达前端的事件，并修改push_status
PUT     /api/v1/events/{event_id}/read             确认事件已读
GET     /api/v1/events/search                      查找设备的事件，可增加查询参数
POST    /api/v1/events/{event_id}/refresh-url      为某个设备某个事件语音续时访问链接
websocket  实时推送最新事件
        /ws/v1/events/{device_id}/new


设备页面：
GET     /api/v1/devices                            展示设备列表获取设备信息
POST    /api/v1/devices/bind                       输入设备id和相应绑定码鉴权绑定设备
GET     /api/v1/devices/search                     按条件查找设备
PUT     /api/v1/devices/{device_id}/info           更新某个设备的信息
DELETE  /api/v1/devices/{device_id}/binding        删除某个设备的绑定
websocket  获取每个设备的在线状态
        /ws/v1/devices/status




统计页面（暂留）：
/api/v1/status/

/---------------------------------------------------------------/
POST    /api/v1/auth/refresh                       用刷新Token自动刷新JWTToken
Content-Type: application/json
{
  "refresh_token": "jwt_refresh_token"
}


成功返回
{
  "code": 0,
  "message": "刷新成功",
  "data": {
    "access_token": "new_jwt_access_token",
    "token_type": "Bearer",
    "expires_in": 7200
  }
}

失败返回

{
  "code": 0,
  "message": "刷新失败，请重新登录",
  "data": null
}
